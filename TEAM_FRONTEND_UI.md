# Team Member 1: Frontend UI and User Experience

## Goal

Make TransitOps look polished, responsive, and demo-ready. This person owns the visual quality of the app and the usability of the main screens.

## Main Files

- `src/styles.css`
- `src/components/Layout.tsx`
- `src/components/KpiCard.tsx`
- `src/components/StatusBadge.tsx`
- `src/pages/Dashboard.tsx`
- `src/pages/Reports.tsx`

## Tasks

1. Improve the dashboard layout.
   - Make KPI cards easy to scan.
   - Make live trips and risk alerts visually clear.
   - Keep the design professional and operations-focused.

2. Improve responsive behavior.
   - Test on mobile width.
   - Make tables scroll cleanly.
   - Make the sidebar and topbar usable on smaller screens.

3. Polish status badges.
   - Use clear colors for `Available`, `On Trip`, `In Shop`, `Suspended`, `Expired`, `Completed`, and `Cancelled`.
   - Keep badge colors readable and consistent.

4. Improve reports visualization.
   - Make the vehicle cost breakdown more presentable.
   - Add better labels and spacing.
   - Optional: add chart library support later if time permits.

5. Add empty states.
   - If there are no trips, drivers, expenses, or maintenance logs, show a clean empty message.

## Deliverable

A visually strong UI that looks good during the final demo and does not break on smaller screens.

## Demo Talking Points

- "The dashboard gives fleet managers a real-time view of vehicles, trips, costs, and risks."
- "The interface highlights blocked vehicles, expired licenses, and maintenance issues before they become operational problems."
